## Confirmed but not fixed

- BSW-002 · high · security/api-contract · flask_security/core.py:1192 · is_locked name and docstring ("True = locked") are inverted relative to all call sites (`if not is_locked(): return False`) and the default `return True`; an app overriding it per the documented contract admits locked users and blocks normal ones
