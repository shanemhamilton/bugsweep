### High
- **B1-1** · architectural · `cmd/dashboard/controller/oauth2.go:215,176-189` — `exchangeOpenId` returns `gjson.GetBytes(body, UserIDPath).String()` with **no empty-string check**. If the IdP userinfo lacks the configured `UserIDPath` (misconfig or attacker-influenced/optional field), `openId==""`. The login path then does `Where("provider=? AND open_id=?", provider, "").First(&bind)` and mints a JWT for the matched row — collapsing identities and enabling login as whichever account first bound an empty openId. **Trigger:** provider whose `UserIDPath` resolves empty. Validate non-empty openId before any bind/login.
- **B1-2** · architectural · `cmd/dashboard/controller/oauth2.go:159-171` — `RTypeBind` looks up an existing bind by `(provider, open_id)` only, then unconditionally sets `bind.UserID = requester` and `Save`s, silently **reassigning an already-bound identity** instead of rejecting the conflict. Combined with B1-1 (empty openId) this forms an account-confusion chain. **Trigger:** complete bind flow for an openId already owned by another user (or the empty-openId row).
- **B4-1** · local · `model/rule.go:82` — `net_all_speed` computes `NetOutSpeed + NetOutSpeed` (outbound counted twice, **inbound ignored**); cf. correct `transfer_all` at :88 (`Out+In`). **Trigger:** any `net_all_speed` alert rule — inbound-heavy servers never alert; outbound trips at half the threshold. Clear logic bug.
